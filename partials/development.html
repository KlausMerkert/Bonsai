Die Bonsai-Simulationssoftware steht unter der GPL v.3 und kann von unserem Repository bei GitHub herunter geladen werden. Wir strukturieren das Repository nach Git-Flow, was bedeutet, dass die neueste stabile Version immer im master-Branch zum Download bereit steht. Die aktuelle Entwicklerversion findest du im Branch develop. Es kann unfertige Features, an denen aktuell entwickelt wird, in Branches mit dem Präfix "feature/" geben. Bitte halte dich an diese Strukturen für deine eigenen Entwicklungen an Bonsai, wenn du vorhast, zum Projekt beizutragen.

Installationsanleitung

Du kannst Bonsai von deinem eigenen Webserver ausliefern. Da die Software komplett in JavaScript geschrieben ist und dieser Code eigentlich nur auf Clientseite ausgeführt wird, hat Bonsai keine besonderen Voraussetzungen an deinen Webserver. Wir haben Bonsai mit Nginx und Apache getestet, aber es wird mit jedem Webserver laufen, der statische Dateien ausliefern kann. Bitte beachte die Angular-Dokumentation, falls du vorhast, das JavaScript zu minimieren. Angular benutzt Variablennamen für dependency injection und hat daher Probleme, wenn der Minifyer die Variablennamen verkürzt.

Wir empfehlen die neueste stabile Version per Git zu installieren: 

$ git clone https://github.com/KlausMerkert/Bonsai.git
Updates können dann ganz leicht mit 
$ git pull
installiert werden.

Du kannst aber auch einfach das .zip-Archiv herunter laden und in deinen Webspace entpacken.

Entwicklersetup

Um an Bonsai zu entwickeln, solltest du einen lokalen Webserver installieren. Bonsai wird nicht richtig funktionieren, wenn du nur index.html mit deinem Browser öffnest, da moderne Browser aus Sicherheitsgründen keine weiteren Dateien von der Festplatte nachladen.

Rufe index.html auf localhost auf, um Bonsai zu starten. Die Unittests kannst du ausführen, indem du jasmine.html ausführst.

Strukturierung des Codes

Bonsai nutzt eine klare Trennung von Models, Views und Controllern (siehe MVC bei Wikipedia). Models sind alle ohne anzeigerelevante Teile im "models/" Ordner implementiert. Wir emulieren Klassen mit Objekt Prototypen, weil JavaScript kein "class"-Schlüsselwort kennt. Ein Blick in den Quelltext der Models wird dir die klassenartige Struktur deutlich machen (Beispiel).

Um die verschiedenen Einzelteile der CPU auch anzuzeigen, benutzen wir Direktiven. Angular bietet die Möglichkeit diese selbstdefinierten HTML-Elemente zu erzeugen und damit gewissermaßen für jedes Einzelteil einen eigenen abgeschlossenen View zu definieren. Jede Direktive hat dafür eigene Controller (Beispiel) und ein HTML-File als View (Beispiel).

Zusätzlich zu den Einzelteilen gibt es einen Controller, der den Rahmen für alle Einzelteile bereit stellt und Busse und Verbindungsleitungen initialisiert. Der zugehörige View definiert die sichtbare und funktionale Struktur der CPU.
